# Первые пробные тренировки ПадлХАБ

Тестовое задание: расписание первых пробных тренировок в визуальном формате турнирной ленты ПадлХАБ.

## Стек

- Next.js 16 App Router и React 19;
- TypeScript и CSS Modules;
- Axios для VivaCRM API;
- nuqs для синхронизации даты и станции с query-параметрами;
- Vitest;
- статический экспорт и GitHub Pages.

## Работа с данными

VivaCRM разрешает браузерные запросы только с домена `padlhub.ru`, поэтому прямой вызов API с GitHub Pages блокируется CORS. Расписание загружается Axios-клиентом во время `next build`, нормализуется и попадает в статический HTML.

GitHub Actions пересобирает сайт каждые три часа. При ошибке API новая версия не публикуется, а последняя успешная остаётся доступной.

Используемые параметры:

- город — Москва;
- направление — `3108`, «Первая пробная тренировка»;
- период — ближайшие 14 дней.

Выбранные дата и станция хранятся в URL, например `?date=2026-07-23&station=...`, поэтому не теряются после перезагрузки и ссылкой можно поделиться.

## Локальный запуск

Требуется Node.js 20.9 или новее.

```bash
npm install
npm run dev
```

Проверки и production-сборка:

```bash
npm run lint
npm run typecheck
npm run test
npm run build
```

Сборка требует доступ к `api.vivacrm.ru` и создаёт статический сайт в `out`.

## Публикация

1. Создать GitHub-репозиторий и отправить проект в ветку `main`.
2. В `Settings → Pages → Build and deployment` выбрать `GitHub Actions`.
3. Запустить workflow вручную или дождаться push в `main`.

`basePath` и `assetPrefix` вычисляются из `GITHUB_REPOSITORY`, поэтому workflow работает и для project Pages, и для репозитория `username.github.io`.

## Ограничения

- Данные обновляются при сборке, а не в реальном времени.
- Оплата, авторизация, запись и лист ожидания не реализованы согласно условию задания.
- Интерфейс повторяет турнирное расписание, но использует данные первых пробных тренировок.

Проект выполнен в рамках трёхчасового таймбокса.
